# Physics

## 목표

물리학적으로 완벽한 시뮬레이터가 아니라, 플레이어가 아래 관계를 명확하게
체감하는 게임 물리를 만든다.

- 잉크가 많을수록 무겁다.
- 무거운 말은 같은 힘으로 덜 움직인다.
- 무거운 말은 쉽게 밀리지 않는다.
- 가벼운 말은 빠르지만 쉽게 밀린다.
- 중심을 치면 직선 이동이 강하다.
- 끝을 치면 회전이 강하다.
- 사용자가 만든 빈 공간은 실제 빈 공간이다.

## 엔진

Planck.js를 사용한다. 게임은 위에서 내려다보는 top-down 구조이므로 중력은
사용하지 않는다.

```js
const world = new planck.World({
  gravity: planck.Vec2(0, 0),
});
```

## Body와 Fixture

말 하나는 하나의 Dynamic Body다. 각 픽셀은 그 Body에 붙은 사각형 Fixture로
만든다.

- 각 픽셀을 별도 Body로 만들지 않는다.
- 내부 빈 공간에는 Fixture를 만들지 않는다.
- 하나의 말에 속한 Fixture끼리는 충돌하지 않는다.
- 말은 충돌해도 조각나지 않는다.
- 초기 구현은 픽셀 하나당 Fixture 하나를 사용한다.

성능 최적화가 필요해질 때만 인접 픽셀 병합을 검토한다. 병합으로 구멍이나 홈이
사라지면 안 된다.

## 질량

기본 밀도는 모든 픽셀에 동일하게 적용한다. Body 질량은 픽셀 수와 Fixture 면적에
따라 자연스럽게 증가해야 한다.

질량 계산은 함수로 분리해, 나중에 선형 관계에서 완만한 곡선으로 바꿀 수 있게
한다. 화면상의 속도를 임의로 보정해 질량감을 만들지 않는다.

## 충격량

발사는 지속 힘이 아니라 순간 충격량으로 처리한다. 같은 드래그 거리에서는 말의
질량과 관계없이 같은 기본 충격량을 적용한다.

드래그 벡터의 반대 방향으로 발사하며, 최대 드래그 거리를 넘으면 충격량이 더
커지지 않는다.

## 회전

충격량은 Body 중심이 아니라 사용자가 누른 실제 World Point에 적용한다.
무게중심에서 먼 지점을 치면 토크가 커지고 회전이 발생한다.

## 고정 시간 간격

물리 루프는 고정 timestep을 사용한다.

- 기본 timestep: `1 / 60`
- `requestAnimationFrame`의 경과 시간을 accumulator에 쌓는다.
- 한 프레임의 최대 substep을 제한한다.

## 정지 판정

모든 말의 선형 속도와 각속도가 기준값 이하인 상태가 일정 시간 유지되면 정지로
판단한다. 이 판정은 게임 규칙용으로 별도 관리한다.

## 탈락 판정

게임판에는 벽이 없다. 초기 MVP에서는 Body의 무게중심이 게임판 경계를 벗어나면
탈락으로 판정한다. Body 삭제는 물리 step 중 즉시 하지 않고 안전한 시점에 처리한다.
